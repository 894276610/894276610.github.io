1.1.1. Workspace 相关命令

  1. Create workspace
   mkdir catkin_ws && cd catkin_ws  
   wstool init src  
   catkin_make      
   source devel/setup.bash
  • catkin_make是在catkin工作区中构建代码的便捷工具。 catkin_make遵循catkin工作区的标准布局
  • source命令功能：使Shell读入指定的Shell程序文件并依次执行文件中的所有语句
  • source命令通常用于重新执行刚修改的初始化文件，使之立即生效，而不必注销并重新登录
  1. Add Repo to Workspace 
    roscd; cd ../src
wstool set repo_name \
--git http://github.com/org/repo_name.git \
--version=kinetic-devel  
wstool up
  • roscd是ros系统中的命令，它可以直接切换到指定目录（ros系统中的软件包）
  • roscd只能切换到那些路径已经包含在ROS_PACKAGE_PATH环境变量中的软件包
    这时候我们需要切换到ros系统的工作空间（catkin_ws）下，通过source命令将软件包添加到环境变量中，即：
    cd ~/catkin_ws
    source ./devel/setup.bash
  1. Resolve Dependencies in Workspace
sudo rosdep init # only once
rosdep update
rosdep install --from-paths src  --ignore-src \
--rosdistro=${ROS_DISTRO} -y

1.1.2. Package 相关命令

  1. Create a package

    catkin_create_pkg package_name [dependencies]

  2. Package folders

    include/package_name c++ header files

src source files
     python libraries in subdirectories
 scripts     python nodes and scripts
 msg,srv,action    Message,Service,and Action Definitions

  3. Release Repo Packages

    catkin_generate_changelog
# review & commit changelogs
catkin_prepare_release
bloom-release --track kinetic --ros-distro kinetic repo_name

  4. Reminders

  • Testable logic
  • Publish diagnostics
  • Desktop dependencies in a separate package

1.1.3. CMakeLists.txt

  1. skeleton(骨架)
cmake_minimum_required(Version 2.8.3)
project(package_name)
find_package(catkin_required)
catkin_package()
  1. package dependencies

To use headers or libraries in a package, or to use a packages’ exported CMake macros, express a build-time dependency:

find_package (catkin required components roscpp)

Tell dependent packages what headers or libraries to pull in when your package is declared as a catkin component:

catkin_package(
   INCLUDE_DIRS include
   LIBRARIES ${project name}
   catkin_depends roscpp
)

Note that any packages note as CATKIN_DEPENDS dependencies also be declared as a <run_depend> in package.xml

  1. Messages,services
    These go after find_package() , but before catkin_package().
    Example:
find_package (CATKIN REQUIRED COMPONENTS message_generation std_msgs)
add_message_files(Files MyMessage.msg)
add_service_files(Files MyService.msg)
generate_messages(DEPENDENCIES std_msgs)
catkin_package(CATKIN_DEPENDS catkin_runtime std_msgs)ww
  1. Build libraries, Excutables
    Goes after the catkin_package() call
add library(${PROJECT_NAME} src/main)
add excutable(${PROJECT_NAME}_node src/main)  
target_link_libraries(${PROJECT_NAME}_node ${catkin_LIBRARIES})
  1. Installation
install (TARGETS ${PROJECT_NAME} DESTINATION ${CATKIN_PACKAGE_LIB_DESTINATION})
install (TARGETS ${PROJECT_NAME}_node DESTINATION ${CATKIN_PACKAGE_BIN_DESTINATION})
install (PROGRAMS scripts/myscript DESTINATION ${CATKIN_PACKAGE_BIN_DESTINATION})
install (DIRECTORY launch DESTINATION ${CATKIN_PACKAGE_SHARE_DESTINATION})

1.1.4. RUNNING SYSTEM

Run ROS using plain:

roscore

Alternatively, roslaunch will run its own roscore automatically if it cannot find one.

roslaunch my_package package_launchfile.launch

suppress the behavior with the --wait flag

  1. Nodes, Topics, Messages
rosnode list  
rostopic list
rostopic echo cmd_vel
rostopic hz cmd_vel
rostopic info cmd_vel
rosmsg show grometry_msgs/Twist
  1. Remote connection
    Master’s ROS Environment
  • ROS_IP or ROS_HOSTNAME set to this machine’s network address
  • ROS_MASTER_URI set to URI that containing that IP or hostname.

Your environment:

  • ROS_IP or ROS_HOSTNAME set to this machine’s network address
  • ROS_MASTER_URI set to URI that from the master.

To debug, check ping from each side to the other, run roswtf on each side.

  1. ROS console
    Adjust using rqt_logger_level and monitor via rqt_console. To enable debug output across sessions, edit the /HOME/.ros/config/rosconsole.config and add a line for your package:
log4j.logger.ros.package_name=DEBUG

And add the following to your sesssion:

export ROSCONSOLE_CONFIG_FILE=$/HOME/.ros/config/rosconsole.config

Use the roslaunch --screen flag to force all the node output to the screen, as if each declared <node> had the output="screen” attribite.

1.2. 启动小海龟

  1. 启动终端 启动ROS MASTER
    roscore
  2. 启动新终端 小海龟仿真器
    rosrun 功能包名 节点名称
    rosrun turtlesim turtlesim_node
  3. 启动新终端 启动海龟控制节点
    rosrun turtlesim turtle_teleop_key

1.3. 分析小海龟话题和服务

  1. 基于qt的可视化工具 rqt graph
  • 可以显示计算图
  • 明白这个系统的全貌
  1. 显示节点相关信息的指令 rosnode

  • 显示节点列表 rosnode list

  • 节点具体信息rosnode info /turtlesim

    • 某个节点具体信息是什么样的
    • 可以看到它发布的订阅的话题
    • 服务配置、主机号等等 3.跟话题相关的参数 rostopic

  • rostopic list 打印当前系统的所有的话题列表

1.4. 使用命令行工具让海龟动起来 （话题通讯）

  • rostopic pub(发布) 话题名 消息结构 消息内容
  • 结果： 海龟只动了一下
rostopic pub /turtle1/cmd_vel geometry_msgs/Twist "linear: 
x: 1.0
y: 0.0 
z: 0.0 
angular: 
x: 0.0 
y: 0.0 
z: 1.0 "
  • 改进方法：重复执行
  • rostopic pub -r(r代表rate 单位Hz) 10
pub -r 10  /turtle1/cmd_vel geometry_msgs/Twist "linear: 
x: 1.0
y: 0.0 
z: 0.0 
angular: 
x: 0.0 
y: 0.0 
z: 1.0 "
  • 查看消息结构到底是怎么样的
    rosmsg show 消息结构
    rosmsg show geometry_msgs/Twist

1.5. 添加新海龟 （服务通讯）

  • 查看所有服务列表
    rosservice list
  • 请求服务
请求产生一个新海龟，设定初始坐标（屏幕左下为原点）
rosservice call /spawn "x: 0.0 
y: 0.0 
theta: 0.0 
name: 'turtle2'"

服务通讯成功在命令行会有一个反馈信息

1.6. 话题记录和复现

  • 话题记录 rosbag record
    -a 表示 all 保存所有的数据
    -O 数据保存成一个压缩包 后边加上文件名
rosbag record -a -O cmd_record

数据会保存在当前终端的默认路径下

  • 话题复现 rosbag play
    重启roscore
    启动仿真器节点
    不启动键盘控制节点
    通过数据包复现指令 （指令复现，起点可能不同） rosbag play 文件名
rosbag play cmd record.bag

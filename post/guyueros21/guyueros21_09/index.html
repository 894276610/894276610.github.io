[学习笔记]古月ROS入门21讲——第九讲

创建工作空间与功能包

  • 创建工作空间与功能包
    • 工作空间
    • 功能包

工作空间

工作空间存放工程文件

  • src (source space)
  • build (build space)
    编译中间文件
    基本不用关心
  • devel (development space)
    编译的库和脚本
  • install (install space) 安装成功之后的位置
  1. 创建工作空间
    创建文件夹->进入文件夹->将当前文件夹变成工作空间属性
    mkdir -p ：递归创建目录，即使上级目录不存在，会按目录层级自动创建目录
    init 操作之后， 会出现CMakeList.txt 文件，初始化成功
mkdir -p ~/catkin_ws（可自定义）/src  
cd ~/catkin_ws/src  
catkin_init_workspace
  1. 编译工作空间、功能包
    回到根目录下->做编译 编译src中的源码 结果： 产生devel, build 两个文件夹
cd ~/catkin_ws/  
catkin_make

如果想要产生install 文件夹

catkin_make install
  1. 设置环境变量
source devel/setup.bash
  1. 检查环境变量
    通过这个环境变量查找所有功能包的路径
echo $ROS_PACKAGE_PATH

功能包

放置ROS源码的最小单元
同一个工作空间不能有同名功能包

  1. 创建功能包
    功能包可能需要依赖其他的功能包
    std_msgs: 是Ros 定义的一些标准的消息结构
    catkin_create_pkg <package_name> [depend1][depend2][depend3]
cd ~/catkin_ws/src  
catkin_create_pkg test_pkg std_msgs rospy roscpp

可以看到生成了 test_pkg 文件夹
cpp 头文件可以放到test_pkg/include 文件夹
源代码放在test_pkg/src 文件夹
CMakeList, package 标识此文件夹是一个功能包

  1. 编译功能包
    回到根目录下
cd ~/catkin_ws/  
catkin_make  
source devel/setup.bash
  • package.xml
    • 版本信息
    • 依赖信息
  • CMakeList.txt
    • 文件编译规则
    • 使用CMake语法，基于gcc编译器
    • 依赖项
